⊕ Welcome to disParity 0.36 Beta ⊕

The new version of disParity is a near-total rewrite of the old utility. It now features a full Windows-based UI. It is no longer necessary to edit your configuration by hand or type any commands on the command line to keep your backup up to date. You can leave disParity running on your system and update your parity backup with a single button press.

Note: command line support is currently disabled in the beta.

Bugs

Every effort has been made to make this new version as reliable and easy to use as possible. However, this is still beta software, so bugs can occur. Your feedback and bug reports will help stabilize it and get it ready for a wider audience. If you encounter any problems with the beta, please report them in the forum. If you need a forum account, please send me an email.

Feedback of any kind on the beta, positive or negative, is welcome. If you don't want to post in the forum, you can email me with any comments, questions, or suggestions you may have.

Requirements

The only requirements for running the disParity beta are Windows XP or later and the .NET Framework 4.0. The installer should detect if you do not have .NET 4.0 installed and take you to the download page automatically.

Upgrading to the beta

If you are a new user, simply install and launch disParity. Follow the instructions in the UI for setting up a new backup configuration.

For existing disParity users, the new UI version of disParity is backwards compatible with previous versions, except for the configuration file, so you should be able to upgrade to the beta without having to rebuild your backup. The new configuration file is no longer meant to be edited by hand. Migrating your configuration over requires the following steps to be performed one time only:

1. Install disParity and launch it from the desktop shortcut.
2. Accept the license agreement.
3. After the main window comes up, press the "Options..."" button.
4. In the Options dialog, press the "Import..." button.
5. Select the location of your old config.txt file and press OK.
6. Your old configuration will be converted to the new format and saved in disParity's application data folder (typically under C:\Users\username\AppData\Local\disParity). The old config.txt file will not be modified.

Important note: during the course of the new beta, parity corruption bugs were discovered in the old command line version of disParity. If your backup was created with version 0.22 or earlier, it is highly recommended that you run a Verify pass (press the "Verify" button in the main window) after upgrading to the new version. The Verify pass will check your parity data for errors and in most cases repair them automatically.
